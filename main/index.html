# LangSmith Alerts：提前捕获生产故障

可靠的应用程序是优质用户体验的基础，而提前捕获故障则是关键。LangSmith 推出了 Alerts 功能，帮助开发者实时监控 LLM 应用和代理。支持基于 错误率、运行延迟 和 反馈评分 等关键指标设置告警。

LLM 应用面临两大挑战：

  1. 对外部服务的依赖：模型提供商、API、数据库等的中断或延迟会严重影响体验。
  2. 输出质量与正确性：提示词或模型的小变化可能导致结果偏差，用户反馈评分可作为早期预警系统。

LangSmith 支持灵活过滤、聚合窗口（5/15 分钟）和阈值设置，并通过 PagerDuty 或自定义 Webhook 集成到现有工作流中。

未来计划包括更多告警类型（如运行计数、Token 使用量）、相对值变化告警（如延迟增加 25%）以及自定义时间窗口。

### 总结

构建可靠的智能体系统的核心难点在于确保大语言模型（LLM）在每一步都能获得适当的上下文。这包括精确控制输入LLM的内容，以及运行适当的步骤生成相关上下文。智能体系统通常由工作流和智能体组成，大多数框架只是提供了智能体的抽象，而非完整的编排工具。这些抽象虽然便于入门，但往往掩盖了对LLM上下文控制的复杂性。

LangGraph 是一个兼具声明式和命令式 API 的编排框架，支持工作流、智能体及两者的混合形式。它提供了短/长期记忆、人类介入、流式处理、调试与可观测性等功能，适用于各种复杂程度的智能体系统。

目前，许多智能体框架仅提供高级抽象，缺乏灵活性，而 LangGraph 提供了低门槛的入门方式，同时支持通过底层功能实现复杂的定制需求。未来，随着模型能力的提升，某些简单任务可能仅需工具调用循环即可完成，但对于大多数企业级应用，结合工作流与智能体的混合系统仍是主流选择。

# Harmonic 创新投资引擎：用 LangGraph 和 LangSmith 提升 VC 效率

Harmonic 是一家专注于早期创业公司发现与分析的引擎，通过整合公共数据和合作伙伴提供的私有数据，帮助风险投资人（VC）更高效地筛选目标企业。借助 LangGraph 和 LangSmith，Harmonic 实现了从简单的数据搜索到复杂投资流程自动化的飞跃。

核心问题

传统方式下，VC 面临海量数据筛选难题，复杂的过滤条件导致效率低下。Harmonic 引入自然语言搜索功能，显著缩短了用户找到符合投资需求的初创企业的时间。

解决方案

  1. LangGraph Studio
  • 提供模块化工作流，支持快速调试和迭代。
  • 通过可视化界面，工程师可以直接检查节点状态、修改参数并重新运行，大幅降低开发时间。
  • 模块化设计让子图可复用，如“研究代理”功能被快速集成到每个公司档案中。
  1. LangSmith
  • 提供强大的模型调用跟踪和协作式提示词优化。
  • 支持版本管理和性能评估，确保每次改动都经过严格测试。
  • 加速团队在不同 LLM 模型间的切换和优化，提升整体开发效率。

成果与影响

  • 用户搜索相关性提升，创意查询的成功率提高 30%。
  • 搜索结果生成时间从小时级降至分钟级甚至秒级。
  • 新增市场地图和深度研究功能，结合用户 CRM 数据和公开网络信息，为投资者提供全面洞察。
  • 帮助 VC 更快找到优质项目，并在竞争中占据优势。

通过 LangGraph 和 LangSmith 的支持，Harmonic 成功打造了一个高效的智能投资助手，助力投资人专注于与创始人的深度互动。
