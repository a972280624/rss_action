  • Doubling down on DeepAgents

    ```markdown # 深度代理的强化 两个月前，我们提出了“深度代理”（Deep Agents）的概念，指的是能够执行复杂、开放性任务并在更长时间范围内运作的代理。我们认为这类代理包含四个关键元素：规划工具、文件系统访问权限、子代理以及详细的提示。为此，我们发布了Python包`deepagents`，它集成了这些基础元素，用户只需添加自定义工具和提示即可轻松构建深度代理。 随着0.2版本的发布，我们引入了可插拔后端作为主要更新点，允许用户将任何存储系统作为“文件系统”接入，包括LangGraph状态、LangGraph存储（跨线程持久化）及本地文件系统等。此外，还推出了“复合后端”，使得如长期记忆等功能成为可能。例如，可以使用本地文件系统为基础，在`/memories/`目录下映射至S3支持的虚拟文件系统。 其他改进还包括大工具结果驱逐、对话历史摘要化以及悬空工具调用修复等功能。关于何时选用`deepagents`而非`langchain`或`langgraph`，我们建议根据项目需求来决定。`langgraph`适用于构建工作流与代理的组合；`langchain`适合需要从零开始构建所有提示和工具的场景；而`deepagents`则更适合开发具有自主性且运行时间较长的代理，因为它内置了诸如规划工具、文件系统等特性。 ```
