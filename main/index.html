    # LangSmith Alerts：提前捕获生产故障 可靠的应用程序是优质用户体验的基础，而提前捕获故障则是关键。LangSmith 推出了 **Alerts** 功能，帮助开发者实时监控 LLM 应用和代理。支持基于 **错误率**、**运行延迟** 和 **反馈评分** 等关键指标设置警报。 ## 为什么主动监控很重要？ LLM 应用面临两大挑战： 1. **对外部服务的依赖**：模型提供商、API、数据库等依赖可能导致性能下降。 2. **输出质量和正确性**：提示词或输入变化可能影响结果质量，用户反馈评分可作为早期预警。 ## LangSmith Alerts 概览 支持以下指标的警报： - 错误数量和比率 - 平均延迟 - 平均反馈评分 可通过筛选器聚焦特定运行子集，并设置聚合窗口（5 或 15 分钟）和阈值。警报集成到现有工作流中，支持 PagerDuty 或自定义 Webhook（如 Slack 通知）。 [立即开始设置警报](https://docs.smith.langchain.com/observability/how_to_guides/alerts?ref=blog.langchain.dev) ## 未来计划 - 新增警报类型：运行计数、LLM Token 使用量 - 支持相对值变化警报（如延迟激增 25%） - 自定义时间窗口警报 有建议？加入 [LangChain Slack 社区](https://www.langchain.com/join-community?ref=blog.langchain.dev) 反馈您的想法！
