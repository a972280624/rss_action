# LangSmith Alerts：提前捕获生产故障

可靠的应用是优质用户体验的基础，而提前捕获故障则是关键。LangSmith 推出了 Alerts 功能，帮助开发者实时监控 LLM 应用和智能体的表现。

核心功能

支持基于以下指标设置告警：

  1. 错误率
  2. 运行延迟
  3. 反馈评分

通过灵活的过滤器，可针对特定模型、工具调用或运行类型进行监控，并设置聚合窗口（5 或 15 分钟）和阈值。告警可集成到 PagerDuty 或通过自定义 Webhook 发送到 Slack 等工具。

为何需要主动监控？

  • 外部服务依赖：LLM 应用常依赖多个 API、数据库等，延迟或中断会影响体验。
  • 质量与正确性：提示词、模型或输入的变化可能导致输出质量下降，反馈评分告警可及时预警。

未来计划

  • 新增告警类型：运行次数、Token 使用量。
  • 支持相对值告警（如延迟激增 25%）。
  • 自定义时间窗口。

立即开始设置告警 或加入 LangChain Slack 社区 提供反馈！

# Trellix 使用 LangGraph 和 LangSmith 提升日志解析效率

Trellix 是一家领先的网络安全公司，通过使用 LangSmith 和 LangGraph Studio 开发了内部应用 Sidekick，显著提升了运营效率。Sidekick 自动化了繁琐的日志解析和脚本编写任务，将日志解析时间从 数天缩短到几分钟，大幅减少了客户请求积压。

问题背景

Trellix 面临的主要挑战是客户请求的积压和日志解析效率低下。传统上，每个请求需要开发人员花费 2-3 天时间手动解析日志和编写集成代码，导致客户体验不佳。

解决方案

通过 LangGraph 的模块化工具和 LangGraph Studio 的可视化功能，Trellix 构建了一个结构化的日志解析流程：

  • 自动生成未知日志格式的解析器。
  • 加速插件和集成开发，从多天缩短到半天完成。
  • 使用 LangSmith 监控和优化 AI 模型性能，提升递归率和文档检索率等关键指标。

成果与影响

  • 工程师效率提升：日志解析时间从数天减少到几分钟。
  • 客户满意度提高：加快了请求处理速度，减少了积压。
  • 透明性增强：通过 LangGraph Studio 可视化 AI 工作流，帮助非技术利益相关者理解模型逻辑。
  • 未来扩展计划：Trellix 计划将 Sidekick 的能力扩展至外部合作伙伴，进一步推广 AI 驱动的解决方案。

结论

通过 LangSmith 和 LangGraph 的支持，Trellix 成功解决了运营瓶颈，不仅提高了内部效率，还增强了客户体验，为未来的创新奠定了基础。

Harmonic 是一家初创企业发现引擎，通过整合公开数据和合作伙伴的私有数据，帮助风险投资人（VCs）发现符合其投资标准的初创公司。利用 LangGraph 和 LangSmith，Harmonic 提升了投资流程效率，实现了自动市场地图、研究报告和对话交互功能，使投资人能够更快筛选并赢得优质交易。
