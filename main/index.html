# LangSmith Alerts：提前捕获生产故障

可靠的应用是优质用户体验的基础，而提前捕获故障是关键。LangSmith 推出了 Alerts 功能，帮助开发者实时监控 LLM 应用和代理。支持基于 错误率、运行延迟 和 反馈评分 设置警报。

为什么主动监控很重要？

  1. 对外部服务的依赖
    LLM 应用依赖多个外部服务（如模型提供商、API、数据库等），任何中断或延迟都会影响用户体验。
  2. 质量与正确性
    LLM 输出的质量可能因提示、模型或输入的变化而不稳定，反馈评分警报能及时发现质量问题。

LangSmith Alerts 概览

支持以下指标的警报：

  • 错误数量和比率
  • 平均延迟
  • 平均反馈评分

可设置过滤器、聚合窗口（5/15 分钟）和阈值，并通过 PagerDuty 或自定义 Webhook 集成到现有工作流中。

下一步计划

未来将增加更多功能，例如运行计数、LLM Token 使用量警报，以及相对变化和自定义时间窗口的警报。

    确保大语言模型（LLM）在每一步都获得适当的上下文，包括精确控制输入内容和生成相关上下文的步骤。

    智能体系统通常结合了“工作流”和“智能体”两种模式。工作流提供更高的可控性和可靠性，而智能体则更具灵活性。

    大多数智能体框架仅提供抽象，而非灵活的编排层，这可能掩盖上下文传递的细节，增加开发难度。

  4. LangGraph 的特点
    LangGraph 是一个兼具声明式和命令式 API 的编排框架，支持智能体和工作流的混合使用，并提供短/长期记忆、容错、人类参与等生产级功能。

  5. 未来趋势与争议
    随着模型能力提升，简单工具调用循环可能满足部分需求，但大多数企业仍需定制化解决方案。OpenAI 的观点被批评为混淆了“声明式 vs 命令式”与“智能体 vs 工作流”的概念。
