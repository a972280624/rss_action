    # 总结：如何将Claude Code转变为特定领域的编程代理 本文探讨了如何优化AI编程助手，使其更擅长编写针对特定领域库（如LangGraph和LangChain）的代码。作者Aliyan Ishfaq分享了他们在尝试多种上下文工程技术后的发现，强调高质量、浓缩的信息加上工具访问更多细节的能力能产生最佳结果。 主要实验包括四个不同的Claude配置测试： - **Claude Vanilla**：未修改的标准版。 - **Claude + MCP**：连接到MCPDoc服务器以获取文档访问权限。 - **Claude + Claude.md**：包含详细指南文件的版本。 - **Claude + MCP + Claude.md**：结合上述两者的优势。 研究发现，虽然直接给AI提供原始文档访问权并未显著提升其性能，但使用精心准备的`Claude.md`文件作为基础，并允许其在需要时访问具体部分的文档，则可以大幅提高效果。这种方式不仅成本效益更高，而且在任务完成度和代码质量上表现更好。 关键结论指出，对于希望定制化AI编码助手的团队来说，创建一个精炼的指导文件（如`Claude.md`），专注于核心概念、独特功能及常见陷阱，是性价比最高的方法。而当此文件与能够智能检索相关文档片段的工具结合使用时，可以获得最佳结果。
