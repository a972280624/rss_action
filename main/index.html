    # Webtoon Entertainment 使用 LangGraph 构建叙事理解 AI **Webtoon Entertainment** 是全球数字娱乐公司，以开创垂直滚动漫画格式闻名，运营 **WEBTOON** 和 **Wattpad** 两大平台，连接全球用户与多样化创作者内容，并将原创 IP 改编为影视作品。 为了处理海量内容，Webtoon 开发了 **Webtoon Comprehension AI (WCAI)**，利用 **LangGraph** 的多智能体工作流，实现对角色、叙事和视觉/文本内容的深度理解。WCAI 已部署于营销、翻译和推荐团队，通过自然语言查询提供即时洞察，减少手动操作。 ## 技术细节 WCAI 基于混合架构，结合 **Vision-Language Models (VLMs)** 和 **LangGraph** 的节点化工作流设计，支持动态路由和领域知识注入。核心工作流包括： 1. **角色识别**：提取角色信息并构建结构化档案。 2. **说话人识别**：分析对话气泡并归因到角色。 3. **叙事提取**：生成情节摘要和情感分析。 4. **业务洞察生成**：为不同团队提供定制化输出，如关键词和高亮场景。 ## 为何选择 LangGraph？ - **可控性**：满足生产环境对透明性和可靠性的需求。 - **生产就绪**：轻松部署 API，集成 **LangSmith** 提供调试和追踪能力。 ## 成果与展望 WCAI 显著减少了手动阅读的工作量（降低 70%），提升了团队效率。未来计划包括系统性评估、人机交互优化及外部数据整合。通过 WCAI 和 LangGraph，Webtoon 正在推动叙事理解的规模化创新。

    # LangSmith Alerts：提前捕获生产故障 可靠的应用程序是优质用户体验的基础，而提前发现故障则是关键。LangSmith 推出了 **Alerts** 功能，帮助开发者实时监控 LLM 应用和代理的表现。 ## 核心功能 - 支持基于 **错误率**、**运行延迟** 和 **反馈评分** 设置警报。 - 提供强大的过滤器，专注于特定运行子集（如模型、工具调用或运行类型）。 - 可配置聚合窗口（5 或 15 分钟）和阈值，适应不同需求。 - 集成到现有工作流，支持通过 PagerDuty 或自定义 Webhook（如 Slack）接收通知。 ## 为何主动监控重要？ 1. **对外部服务的依赖**：LLM 应用通常依赖多个外部服务（如 API、数据库等），任何延迟或中断都会影响用户体验。 2. **质量与正确性**：LLM 输出的质量可能因提示、输入或模型变化而波动，反馈评分警报能快速捕捉问题。 ## 未来计划 - 增加更多警报类型（如运行计数、Token 使用量）。 - 支持相对值警报（如延迟增加 25%）。 - 自定义时间窗口的警报。 [立即开始设置警报](https://docs.smith.langchain.com/observability/how_to_guides/alerts?ref=blog.langchain.dev) 并加入 [LangChain Slack 社区](https://www.langchain.com/join-community?ref=blog.langchain.dev) 提供反馈！
