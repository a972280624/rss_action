  • In software, the code documents the app. In AI, the traces do.

    ```markdown 在传统软件中，代码是应用行为的“文档”，决策逻辑明确且可追溯。而在AI智能体（Agent）中，代码仅是框架，真正的决策由模型在运行时完成，具有非确定性。因此，**源代码不再是系统行为的“真相”**，取而代之的是**执行轨迹（Traces）**。 Traces记录了Agent每一步的推理、工具调用及决策原因，成为理解、调试、测试、优化和监控系统的核心依据。开发者需通过分析轨迹来定位问题、评估质量、优化性能，并实现团队协作与产品洞察。缺乏对Traces的良好可观测性，构建Agent就如同“盲目驾驶”。 **结论：从“读代码”转向“看轨迹”，是开发AI Agent的关键范式转变。** ```
