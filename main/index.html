# LangSmith Alerts：提前捕获生产故障

为了提供卓越的用户体验，可靠的应用程序至关重要。LangSmith 推出了 Alerts 功能，帮助开发者实时监控 LLM 应用和代理，提前发现问题，避免影响用户。

核心功能

  • 支持的关键指标：错误率、运行延迟、反馈评分。
  • 灵活配置：通过过滤器聚焦特定运行子集（如模型、工具调用类型）。
  • 灵敏度调整：设置聚合窗口（5或15分钟）和阈值。
  • 集成通知：支持 PagerDuty 或自定义 Webhook（如 Slack 通知）。

为什么主动监控重要？

  1. 外部服务依赖：LLM 应用依赖多个服务（如 API、数据库），这些服务的故障或延迟会显著影响用户体验。
  2. 质量与正确性：LLM 输出可能因提示、模型或输入变化而波动，反馈评分警报可及时捕捉质量下降。

未来计划

  • 新增更多警报类型（如运行计数、Token 使用量）。
  • 支持相对值变化警报（如延迟激增 25%）。
  • 自定义时间窗口警报。

## 如何思考 Agent 框架

构建可靠的 Agent 系统的核心难点在于确保大语言模型（LLM）在每一步都能获得适当的上下文。这包括精确控制输入 LLM 的内容，以及运行生成相关上下文的步骤。

关键点总结

  1. Agent 系统的组成
    Agent 系统既包含工作流（Workflows），也包含 Agent，以及介于两者之间的各种形式。

  2. 框架的本质
    大多数 Agent 框架并非声明式或命令式编排框架，而只是提供了一组 Agent 抽象。这些抽象虽然易于上手，但可能隐藏了对 LLM 上下文的精确控制。

  3. LangGraph 的定位
    LangGraph 是一个编排框架，结合了声明式和命令式 API，并在其基础上构建了一系列 Agent 抽象。它支持动态和静态系统设计，同时提供了短时记忆、长时记忆、人类介入、流式处理等生产级功能。

  4. Agent 与 Workflow 的对比

  • Workflow：更可预测，适合定义明确的任务。
  • Agent：更具灵活性，适合需要大规模动态决策的场景。
    实际生产中的 Agent 系统通常是两者的结合。

  1. OpenAI 的误区
    OpenAI 的指南中存在一些误导性观点，例如将声明式与非声明式框架对立，或将 Agent 和 Workflow 对立。实际上，Agent 系统的设计需根据具体任务需求权衡灵活性与可控性。

  2. 未来展望
    随着模型能力的提升，简单工具调用的 Agent 可能会满足部分场景需求，但对于复杂任务，仍然需要精细的工作流控制和上下文管理。

结论

无论是 Agent 还是 Workflow，所有形式的 Agent 系统都可以从一组通用特性中受益，例如短时记忆、长时记忆、故障容忍、调试与可观测性等。LangGraph 提供了一个灵活的编排框架，既能支持高级抽象，又能满足底层控制需求，适用于各种复杂场景。

# Harmonic 利用 LangGraph 和 LangSmith 构建投资智能助手

Harmonic 是一家专注于初创企业发现的引擎，通过整合公共数据与合作伙伴的私有数据，为风险投资者（VCs）提供深度洞察和工作流工具。借助 LangGraph 和 LangSmith，Harmonic 实现了从初创企业筛选到投资决策支持的全流程优化。

问题：发现最具潜力的初创企业

早期初创企业的复杂生态让 VCs 难以高效定位目标。传统搜索方式需要用户在数百个字段中组合筛选条件，耗时且易错过优质标的。Harmonic 引入自然语言搜索和优化功能，显著缩短了用户找到符合投资主题的最佳初创企业的时间。

解决方案

  1. LangGraph Studio
  • 提供模块化开发框架，支持快速构建和调试智能工作流。
  • 工程师可通过可视化界面直接检查节点状态，减少调试时间，并灵活实验不同模型和指令。
  • 模块化设计使“研究代理”等功能快速扩展至平台其他部分，节省投资者在尽职调查中的时间。
  1. LangSmith
  • 提供强大的提示版本管理和协作环境，支持团队对模型调优和提示迭代。
  • 无缝集成执行跟踪和评估功能，确保每次变更都可测试，提升开发信心。

成果

  • 用户从搜索到获得有效结果的时间从数小时缩短至不到一分钟。
  • 积极搜索结果提升 30%，新增即时市场地图和综合研究功能。
  • 投资者能更快锁定优质交易，专注于与创始人的深度互动。

Harmonic 的创新技术帮助 VCs 在竞争激烈的投资市场中占据先机。
