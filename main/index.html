# LangSmith Alerts：提前捕获生产故障

为了提供卓越的用户体验，可靠的应用程序至关重要。LangSmith 推出了实时警报功能，帮助开发者监控 LLM 应用和代理，提前发现问题。通过设置关键指标（如错误率、运行延迟和反馈评分）的警报，团队可以更快地响应潜在问题。

为什么主动监控很重要？

  1. 对外部服务的依赖：LLM 应用通常依赖多个外部服务（如模型提供商、API 和数据库），任何中断或延迟都会影响用户体验。
  2. 质量和正确性：LLM 输出的质量不稳定，小的变化可能导致结果偏差，基于用户反馈的警报可及时发现质量问题。

功能概览

  • 支持的警报指标：错误数量/率、平均延迟、平均反馈评分。
  • 可按模型、工具调用等条件过滤。
  • 灵活设置聚合窗口（5 或 15 分钟）和阈值。
  • 集成 PagerDuty 或自定义 Webhook（如 Slack）。

# 关于智能体框架的思考总结

本文探讨了构建可靠智能体系统的核心挑战及不同智能体框架的特点。以下是主要内容的总结：

  1. 核心挑战
    构建可靠的智能体系统的关键在于确保大语言模型（LLM）在每一步都能获得适当的上下文。这包括精确控制输入内容以及生成相关上下文的步骤。

  2. 智能体系统的组成
    智能体系统由工作流（Workflows）和智能体（Agents）组成，两者之间存在多种组合形式。工作流强调确定性，而智能体则更灵活，允许模型动态决策。

  3. 智能体框架的分类

  • 工作流 vs 智能体：工作流适合确定性任务，智能体适合需要灵活性的任务。
  • 声明式 vs 命令式：LangGraph 是一种兼具声明式和命令式特性的编排框架。
  • 智能体抽象：许多框架提供智能体抽象以简化开发，但可能掩盖上下文控制的复杂性。

  1. LangGraph 的特点
    LangGraph 是一个事件驱动的编排框架，支持声明式、命令式和事件驱动 API，适用于构建各种智能体系统。它提供了短时记忆、长时记忆、人类介入、流式处理等生产级功能。

  2. 常见问题与讨论

  • 框架的价值：除了智能体抽象，框架还提供存储、容错、调试等功能，适用于智能体和工作流。
  • 模型改进的影响：随着模型性能提升，某些任务可能只需要简单的工具调用循环，但仍需上下文控制。
  • OpenAI 的误解：OpenAI 的观点混淆了“声明式 vs 命令式”和“工作流 vs 智能体”，忽略了编排层的重要性。
  1. 结论
    智能体系统的核心挑战是确保 LLM 获得适当上下文。无论是智能体还是工作流，都需要强大的编排框架支持。LangGraph 提供了灵活的低层编排能力和高层智能体抽象，适用于多种应用场景。

# Harmonic 利用 LangGraph 和 LangSmith 构建投资代理，助力风投聚焦创始人

Harmonic 是一家初创企业发现引擎，通过整合海量公共和私有数据，帮助风险投资（VC）更高效地发现和评估初创公司。借助 LangGraph 和 LangSmith，Harmonic 实现了自动市场地图、研究报告和对话式交互，优化了投资流程。

问题：发现最令人兴奋的初创企业

早期初创企业的复杂生态让 VCs 难以快速找到符合其投资标准的目标。传统的搜索方式耗时且繁琐，难以满足用户需求。Harmonic 引入自然语言搜索和筛选功能，显著缩短了用户找到最佳匹配初创企业的时间。

解决方案：LangGraph 和 LangSmith 的优势

  1. LangGraph Studio 提供模块化工作流和可视化调试工具，极大提高了开发效率。
  2. LangSmith 支持协作式提示工程和模型性能跟踪，确保快速迭代和优化。
  3. 模块化框架使 Harmonic 能够快速扩展功能，例如“研究代理”，为每个公司档案提供深度分析。

成果与影响

  • 用户搜索相关性提升，时间从数小时缩短至 1 分钟以内。
  • 积极搜索结果增加 30%。
  • 新功能如即时市场地图和综合研究能力，增强了投资决策支持。

Harmonic 帮助投资者更快找到、选择并赢得最佳交易，成为风投领域的重要工具。
